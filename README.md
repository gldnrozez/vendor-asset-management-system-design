# Inventory Management System Design

## Executive Summary
[cite_start]This repository documents the comprehensive system analysis and technical blueprint for the Vendor Asset Management Information System at PT Pertamina Hulu Rokan[cite: 39]. [cite_start]The project focuses on root-cause analysis and architectural design to ensure seamless integration between vendor data and internal inventory[cite: 18, 59].

## System Architecture & Design Deliverables
The project demonstrates a high-level of technical proficiency by executing the system design lifecycle through the following artifacts:

* [cite_start]**Ishikawa (Fishbone) Diagram:** Conducted a root-cause analysis to identify operational bottlenecks and mitigate organizational risks before system design[cite: 25, 59].
* [cite_start]**Use Case Diagram:** Defined functional requirements and user interactions for core business processes to align with stakeholder objectives.
* [cite_start]**Activity Diagram:** Mapped 8 core business processes into systematic workflows to eliminate data redundancy and enhance operational efficiency[cite: 18, 59].
* **Sequence Diagram:** Detailed the logic of system interactions over time, ensuring precise communication between technical components and data sources.
* [cite_start]**Class Diagram:** Architected a structured database schema comprising 11 interconnected tables to ensure 100% data integrity and accountability.

## The STAR Approach

[cite_start]**Situation:** Requirement for a standardized technical blueprint to manage complex business processes and synchronize vendor assets with internal inventory at ICT Function PHR Zone 4[cite: 39, 41].

[cite_start]**Task:** To engineer a comprehensive system architecture using the **Waterfall methodology** that bridges the gap between technical development and strategic business objectives[cite: 18, 54].

**Action:** 1. Performed deep-dive root-cause analysis using the **Ishikawa** method to identify system requirements.
2. [cite_start]Executed the system design lifecycle by mapping workflows through **UML Diagrams** (Use Case, Activity, and Sequence)[cite: 19, 42].
3. [cite_start]Designed a robust database architecture with 11 interconnected tables to prevent data redundancy.

[cite_start]**Result:** Delivered a standardized technical blueprint for 8 core business processes, optimizing data workflows and providing a clear execution roadmap for the development team[cite: 18, 41].
